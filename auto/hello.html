What is KV Cache?

In Large Language Models (LLMs), the generation of text happens autoregressively—meaning the model predicts one token at a time, using the previously generated tokens as input for the next one.

The Problem: Without a cache, for every new token generated, the model has to re-calculate the mathematical representations (Keys and Values) for all previous tokens. This is redundant and incredibly slow as the sequence grows.

2. How KV Cache Works

The KV Cache stores the Key (K) and Value (V) vectors for every token in the sequence. These are the components used in the Self-Attention mechanism.

  • Prefill Phase: The initial prompt is processed once, and all K/Vs are stored.
  • Decoding Phase: For each new token, the model only computes the K/V for the current token and retrieves the rest from the cache.

The model skips computing history and only calculates the "New Token" K/V, then combines it with the existing cache.

3. The Trade-off: Memory vs. Speed

While KV Cache dramatically speeds up generation (making it possible to have real-time chat), it consumes a significant amount of VRAM (Video RAM).

Feature Without KV Cache With KV Cache
Computation Quadratic / Redundant Linear / Efficient
Memory Usage Low High (Scales with sequence length)
Inference Speed Very Slow Very Fast

Summary

KV Cache is essentially a "memory" for the Attention mechanism. Instead of reading the whole book every time you want to write the next word, you keep a notepad (the cache) of the important summaries (Keys and Values) so you can keep writing without looking back.
